# LLM UI GENERATION GUIDELINES — FLAT DESIGN

This document defines the global design rules that the language model (LLM) must follow when generating user interfaces for web applications. 
These rules standardize typography, color usage, layout, components, and interaction patterns according to Flat Design principles.

The instructions in this document should guide ALL UI generation unless explicitly overridden by the system.

--------------------------------------------------
1. DESIGN PRINCIPLES
--------------------------------------------------

The UI must follow Flat Design principles:

- Clean and minimal visual appearance
- No skeuomorphic or realistic effects
- Avoid heavy shadows, bevels, or 3D effects
- Prefer solid colors instead of gradients
- Maintain strong visual hierarchy
- Keep interfaces readable and accessible
- Use consistent spacing and grid layout
- Components must be simple and predictable

Goal:
Create modern, professional, and scalable web interfaces suitable for dashboards, SaaS systems, fintech applications, and administrative tools.

--------------------------------------------------
2. TYPOGRAPHY
--------------------------------------------------

Typography must always use modern sans-serif fonts.

Preferred fonts:

- Inter
- Roboto
- Open Sans

Font hierarchy:

Size: 32px
Weight: 700

H2
Size: 24px
Weight: 600

H3
Size: 20px
Weight: 600

Body Text
Size: 16px
Weight: 400

Small Text
Size: 14px
Weight: 400

Label
Size: 12px
Weight: 500

Additional rules:

- Line height between 1.4 and 1.6
- Maintain high readability
- Avoid decorative fonts
- Maintain consistent hierarchy

--------------------------------------------------

--------------------------------------------------

Primary Button

Ghost Button

Background: none
Border: none

--------------------------------------------------

--------------------------------------------------

Icons must be simple and minimal.

Recommended style:

- outline icons
- flat icons

Icon sizes:

16px
20px
24px

Lucide
Heroicons

--------------------------------------------------
8. LAYOUT STRUCTURE
--------------------------------------------------

Applications should follow a standard dashboard layout.

Typical layout structure:

Sidebar Navigation
Top Header
Main Content Area

Recommended dimensions:

Sidebar width: 240px
Header height: 64px
Content padding: 24px

Content should be organized using cards and sections.

--------------------------------------------------
9. CARDS
--------------------------------------------------

Cards are the main container for UI content.

Card style:

Background: white
Border: 1px solid #E5E7EB
Border radius: 8px
Padding: 16px

Avoid heavy shadows.

--------------------------------------------------
10. TABLES
--------------------------------------------------

Table structure:

Header background
#F9FAFB

Row border
#E5E7EB

Row hover
#F3F4F6

Cell padding

12px 16px

Tables must remain readable and structured.

--------------------------------------------------
11. FORM INPUTS
--------------------------------------------------

Input style:

Height: 40px
Border: 1px solid #D1D5DB
Border radius: 6px
Padding: 8px 12px

Focus state:

Border color: primary color
Outline: none

Forms must be clear and accessible.

--------------------------------------------------
12. SHADOWS
--------------------------------------------------

Flat design minimizes shadows.

If necessary, use very light shadows:

box-shadow: 0 1px 2px rgba(0,0,0,0.05)

Avoid large elevation effects.

--------------------------------------------------
13. ANIMATIONS
--------------------------------------------------

Animations must be minimal.

Transition standard:

transition: 0.2s ease

Allowed usage:

- hover effects
- dropdown menus
- UI state changes

Avoid excessive animations.

--------------------------------------------------
14. STANDARD APPLICATION STRUCTURE
--------------------------------------------------

When generating dashboards or admin panels, prefer:

Sidebar navigation
Top header
Dashboard cards
Charts
Tables
Forms

This structure is typical for:

- SaaS platforms
- financial systems
- admin dashboards
- enterprise tools

--------------------------------------------------
15. GLOBAL GENERATION RULES FOR THE LLM
--------------------------------------------------

When generating UI code or layouts, the LLM must:

1. Use sans-serif fonts
2. Maintain consistent spacing
3. Prefer solid colors
4. Avoid visual noise
5. Ensure accessibility and readability
6. Keep components simple
7. Follow the defined color palette
8. Follow the defined layout patterns

These rules act as the base design system for all generated interfaces.
